---
title: 호스팅 서비스란?
---

이 섹션에서는 [Hosted Service.](https://thegraph.com/hosted-service/)라고도 알려진 호스팅 서비스에 대한 서브그래프 배포를 알아보겠습니다. 참고로 호스팅 서비스는 곧 종료되지 않습니다. 탈중앙화 네트워크와 어느정도 동등한 기능에 도달하면 호스팅 서비스를 점차적으로 중단할 예정입니다. 호스팅 서비스에 배포된 서브그래프는 [여기](https://thegraph.com/hosted-service/) 에서 계속 사용할 수 있습니다.

호스팅 서비스에 계정이 없는 경우 Github 계정으로 가입할 수 있습니다. 인증이 완료되면 UI를 통해 서브그래프를 만들고 터미널에서 배포할 수 있습니다. 그래프 노드는 메인넷 외에도 수많은 이더리움 테스트넷(Rinkeby, Ropsten, Kovan)을 지원합니다.

## 서브그래프 생성하기

먼저 [여기](/developer/define-subgraph-hosted)의 지침에 따라 Graph CLI를 설치하십시오. `graph init --product hosted service`를 전달하여 서브그래프를 만듭니다.

### 기존 컨트랙트로 부터

이더리움 메인넷이나 테스트넷 중 하나에 이미 스마트 컨트랙트를 구축한 경우, 이 컨트랙트의 새 서브그래프를 부트스트래핑하는 것은 호스팅된 서비스를 시작하는 좋은 방법이 될 수 있습니다.

이 명령을 사용하여 기존 컨트랙트의 모든 이벤트를 인덱싱하는 서브그래프를 만들 수 있습니다. 이는 [Etherscan](https://etherscan.io/)에서 컨트랙트 ABI를 가져오려고 시도할 것입니다.

```sh
graph init \
  --product hosted-service
  --from-contract <CONTRACT_ADDRESS> \
  <GITHUB_USER>/<SUBGRAPH_NAME> [<DIRECTORY>]
```

또한 다음과 같은 선택적 인수를 사용할 수 있습니다. Etherscan에서 ABI를 가져올 수 없는 경우 로컬 파일 경로를 다시 요청합니다. 명령어에 선택적 인수가 없는 경우 대화형 양식을 사용합니다.

```sh
--network <ETHEREUM_NETWORK> \
--abi <FILE> \
```

The `<GITHUB_USER>` in this case is your GitHub user or organization name, `<SUBGRAPH_NAME>` is the name for your subgraph, and `<DIRECTORY>` is the optional name of the directory where graph init will put the example subgraph manifest. The `<CONTRACT_ADDRESS>` is the address of your existing contract. `<ETHEREUM_NETWORK>` is the name of the Ethereum network that the contract lives on. `<FILE>` is a local path to a contract ABI file. **Both --network and --abi are optional.**

### 예제 서브그래프로 부터

`graph init`가 지원하는 두 번째 모드는 예제 서브그래프에서 새 프로젝트를 만드는 것입니다. 다음 명령은 이 작업을 수행합니다.

```
graph init --from-example --product hosted-service <GITHUB_USER>/<SUBGRAPH_NAME> [<DIRECTORY>]
```

예제 서브그래프는 사용자 아바타를 관리하고, 아바타가 생성되거나 업데이트될 때마다 `NewGravatar` 또는 `UpdateGravatar` 이벤트를 방출하는 Dani Grant의 Gravity 컨트랙트에 기반을 두고 있습니다. 서브그래프는 `Gravatar` 앤타타를 그래프 노드 저장소에 기록하고 이벤트에 따라 업데이트되도록 하여 이러한 이벤트를 처리합니다. [서브그래프 매니페스트](/developer/create-subgraph-hosted#the-subgraph-manifest)로 이동하여 스마트 컨트랙트에서 주의를 기울여야 할 이벤트, 맵핑 등에 더 잘 파악해보시기 바랍니다.
