---
title: Subgraph Studio에 서브그래프 배포하기
---

Subgraph Studio에 서브그래프를 배포하는 것은 매우 간단합니다. 다음 단계를 수행할 수 있습니다:

- The Graph CLI (yarn 및 npm 모두 동일) 설치
- Subgraph Studio에서 서브그래프 생성
- CLI에서 계정 인증
- Subgraph Studio에 서브그래프 배포

## Graph CLI 설치

우리는 동일한 CLI를 사용하여 [호스팅 서비스](https://thegraph.com/hosted-service/)와 [Subgraph Studio](https://thegraph.com/studio/)에 서브그래프를 배포하고 있습니다. 다음은 graph-cli를 설치하는 명령입니다. 이는 npm 또는 yarn을 사용하여 수행할 수 있습니다.

**yarn과 함께 설치합니다:**

```bash
yarn global add @graphprotocol/graph-cli
```

**npm과 함께 설치합니다:**

```bash
npm install -g @graphprotocol/graph-cli
```

## Subgraph Studio에서 서브그래프 생성

실제 서브그래프를 배포하기 전에 [Subgraph Studio](https://thegraph.com/studio/)에서 서브그래프를 생성해야 합니다. 이에 대해 자세히 알아보려면 [Studio 문서](/deploying/subgraph-studio)를 읽는 것이 좋습니다.

## 서브그래프 초기화

Subgraph Studio에서 서브그래프가 생성되면 다음 명령을 사용하여 서브그래프 코드를 초기화할 수 있습니다.

```bash
graph init --studio <SUBGRAPH_SLUG>
```

`<SUBGRAPH_SLUG>` 값은 Subgraph Studio의 서브그래프 상세내역 페이지에서 확인 가능합니다:

![Subgraph Studio - Slug](/img/doc-subgraph-slug.png)

`graph init`를 실행한 후 쿼리할 컨트랙트 주소, 네트워크, abi를 입력하라는 메시지가 표시됩니다. 이렇게 하면 서브그래프에서 작업을 시작하기 위한 몇 가지 기본 코드가 포함된 새 폴더가 로컬 컴퓨터에 생성됩니다. 그런 다음 서브그래프를 마무리하여 예상대로 작동하는지 확인할 수 있습니다.

## Graph Auth

Subgraph를 Subgraph Studio에 배포하기 전에 CLI 내에서 계정에 로그인해야 합니다. 이렇게 하려면 "My Subgraphs" 페이지 또는 서브그래프 세부 정보 페이지에서 찾을 수 있는 배포 키가 필요합니다.

다음은 CLI에서 인증하는 데 사용해야 하는 명령입니다:

```bash
graph auth --studio <DEPLOY KEY>
```

## Subgraph Studio에 서브그래프 배포

준비가 되면, Subgraph Studio에 서브그래프를 배포할 수 있습니다. 이렇게 하면 서브그래프가 탈중앙화 네트워크에는 게시되지 않고, 테스트하고 메타데이터를 업데이트할 수 있는 Studio 계정에만 배포됩니다.

다음은 서브그래프를 배포하는 데 사용해야 하는 CLI 명령입니다.

```bash
graph deploy --studio <SUBGRAPH_SLUG>
```

이 명령을 실행한 후 CLI는 버전 레이블을 요청합니다. 원하는 대로 이름을 지정할 수 있습니다. `0.1` 및 `0.2`와 같은 레이블을 사용하거나 `uniswap-v2-0.1`과 같은 문자를 사용할 수 있습니다. 이러한 레이블은 그래프 탐색기에서 볼 수 있으며 큐레이터가 이 버전에서 신호를 보낼지 여부를 결정하는 데 사용할 수 있으므로, 현명하게 이들을 선택하시길 바랍니다.

일단 배포되면 플레이그라운드를 사용하여 Subgraph Studio에서 서브그래프를 테스트하고, 필요한 경우 다른 버전을 배포하고, 메타데이터를 업데이트하고, 준비가 되면 서브그래프를 그래프 탐색기에 배포할 수 있습니다.
