서브그래프가 배포된 상태에서 [그래프 탐색기](https://thegraph.com/explorer)를 방문하여 쿼리를 실행하고 스키마를 확인하여 서브그래프에 대해 배포된 GraphQL API를 탐색할 수 있는 [GraphiQL](https://github.com/graphql/graphiql) 인터페이스를 엽니다.

아래에 예가 제공되지만 서브그래프의 엔터티를 쿼리하는 방법에 대한 전반적인 내용은 [Query API](/developer/graphql-api)를 참조하시기 바랍니다.

#### 예시

이 쿼리는 매핑이 생성한 모든 카운터를 나열합니다. 우리가 하나만 생성하기 때문에 결과에는 `default-counter`가 하나만 포함됩니다.

```graphql
{
  counters {
    id
    value
  }
}
```

## The Graph 탐색기 사용

탈중앙화 그래프 탐색기에 게시된 각 서브그래프에는 서브그래프 세부 정보 페이지로 이동하고 오른쪽 상단 모서리에 있는 "쿼리" 버튼을 클릭하여 찾을 수 있는 고유한 쿼리 URL이 있습니다. 이는 서브그래프의 고유한 쿼리 URL과 쿼리 방법에 대한 몇 가지 지침을 제공하는 사이드 창을 열 것입니다.

![쿼리 서브그래프 판](/img/query-subgraph-pane.png)

보다시피 이 쿼리 URL은 고유한 API 키를 사용해야 합니다. "API 키" 섹션의 [Subgraph Studio](https://thegraph.com/studio)에서 API 키를 만들고 관리할 수 있습니다. [여기](/studio/subgraph-studio)에서 Subgraph Studio를 사용하는 방법에 대해 자세히 알아보시기 바랍니다.

API 키를 사용하여 서브그래프를 조회하면 GRT로 지불되는 쿼리 수수료가 발생합니다. 요금 청구에 대한 자세한 내용은 [여기](/studio/billing)를 참조하시기 바랍니다.

또한 "Playground" 탭의 GraphQL playground를 사용하여 그래프 탐색기 내의 서브그래프를 쿼리할 수 있습니다.
