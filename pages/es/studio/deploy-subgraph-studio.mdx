---
title: Despliegue de un subgrafo en Subgraph Studio
---

Implementar un Subgraph en Subgraph Studio es bastante simple. Esto te llevará a través de los pasos para:

- Instale The Graph CLI (con yarn y npm)
- Crea su Subgraph en Subgraph Studio
- Autentica tu cuenta desde la CLI
- Implementa un Subgraph en Subgraph Studio

## Instalación de la CLI de Graph

Usamos la misma CLI para implementar subgrafos en nuestro [servicio alojado](https://thegraph.com/hosted-service/) y en el [Subgraph Studio](https://thegraph.com/studio/). Aquí están los comandos para instalar graph-cli. Esto se puede hacer usando npm o yarn.

**Instalar con yarn:**

```bash
yarn global add @graphprotocol/graph-cli
```

**Instalar con npm:**

```bash
npm install -g @graphprotocol/graph-cli
```

## Crea tu Subgrafo en Subgraph Studio

Antes de implementar tu subgrafo real, debez crear un subgrafo en [Subgraph Studio](https://thegraph.com/studio/). Te recomendamos que leas nuestra [documentación de Studio](/studio/subgraph-studio) para obtener más información al respecto.

## Inicializa tu Subgrafo

Una vez que se haya creado tu subgrafo en Subgraph Studio, puedes inicializar el código del subgrafo usando este comando:

```bash
graph init --studio <SUBGRAPH_SLUG>
```

El valor `<SUBGRAPH_SLUG>` se puede encontrar en la página de detalles de tu subgrafo en Subgraph Studio:

![Subgraph Studio - Slug](/img/doc-subgraph-slug.png)

After running `graph init`, you will be asked to input the contract address, network, and ABI that you want to query. Doing this will generate a new folder on your local machine with some basic code to start working on your subgraph. You can then finalize your subgraph to make sure it works as expected.

## Graph Auth

Before being able to deploy your subgraph to Subgraph Studio, you need to login into your account within the CLI. To do this, you will need your deploy key that you can find on your "My Subgraphs" page or your subgraph details page.

Este es el comando que debes usar para autenticarse desde la CLI:

```bash
graph auth --studio <DEPLOY KEY>
```

## Implementación de un Subgraph en Subgraph Studio

Una vez que estés listo, puedes implementar tu subgrafo en Subgraph Studio. Hacer esto no publicará tu subgrafo en la red descentralizada, solo lo implementará en tu cuenta de Studio, donde podrás probarlo y actualizar los metadatos.

Este es el comando CLI que debes usar para implementar tu subgrafo.

```bash
graph deploy --studio <SUBGRAPH_SLUG>
```

After running this command, the CLI will ask for a version label, you can name it however you want, you can use labels such as `0.1` and `0.2` or use letters as well such as `uniswap-v2-0.1`. Those labels will be visible in Graph Explorer and can be used by curators to decide if they want to signal on this version or not, so choose them wisely.

Una vez implementado, puedes probar tu subgrafo en Subgraph Studio usando el playground, implementar otra versión si es necesario, actualizar los metadatos y, cuando estés listo, publicar tu subgrafo en Graph Explorer.
