कोडिंग तुरंत शुरू करने के लिए, [डेवलपर क्विक स्टार्ट पर जाएं](/quick-start/)।

## अवलोकन

एक डेवलपर के रूप में, आपको अपने dapp को बनाने और शक्ति प्रदान करने के लिए डेटा की आवश्यकता होती है। ब्लॉकचेन डेटा को क्वेरी करना और इंडेक्स करना चुनौतीपूर्ण होता है, लेकिन The Graph इस समस्या का समाधान प्रदान करता है।

The Graph पर, आप:

1.  "बनाएँ, डिप्लॉय करें, और The Graph में subgraphs प्रकाशित करें Graph CLI और" [Subgraph Studio](https://thegraph.com/studio/).
2.  मौजूदा subgraphs को क्वेरी करने के लिए GraphQL का उपयोग करें।

### GraphQL क्या है?

- [GraphQL](https://graphql.org/learn/) APIs  के लिए एक क्वेरी भाषा और आपके मौजूदा डेटा के साथ उन क्वेरीज़ को पूरा करने के लिए एक रनटाइम। The Graph सबग्राफ को क्वेरी करने के लिए GraphQL का उपयोग करता है।

### डेवलपर क्रियाएँ

- अन्य डेवलपर्स द्वारा बनाए गए subgraphs को [The Graph Network](https://thegraph.com/explorer) में query करें और उन्हें अपनी dapps में एकीकृत करें।
- विशिष्ट डेटा आवश्यकताओं को पूरा करने के लिए कस्टम सबग्राफ़ बनाएं, जिससे अन्य डेवलपर्स के लिए स्केलेबिलिटी और लचीलापन में सुधार हो सके।
- अपने subgraphs को The Graph Network में तैनात करें, प्रकाशित करें और संकेत दें।

## Subgraph विवरण

### सबग्राफ़ क्या हैं?

एक Subgraph एक कस्टम API है जो ब्लॉकचेन डेटा पर आधारित होता है। यह ब्लॉकचेन से डेटा निकालता है, उसे प्रोसेस करता है, और उसे इस तरह से संग्रहित करता है कि उसे GraphQL के माध्यम से आसानी से क्वेरी किया जा सके।

एक Subgraph मुख्य रूप से निम्नलिखित फ़ाइलों से बना होता है।

- `subgraph.yaml`: यह YAML फ़ाइल [subgraph manifest](/developing/creating-a-subgraph/#the-subgraph-manifest) को शामिल करती है।
- `subgraph.graphql`: एक ग्राफक्यूएल स्कीमा जो कौन सा डाटा आपके सुब-ग्राफ ेमिन स्टोर करना है इसको प्रभासित करती है और बताती है कि उसे [GraphQL](/developing/creating-a-subgraph/#the-graphql-schema)के द्वारा कैसे क्वेरी करना है|
- `mappings`: यह [AssemblyScript](https://github.com/AssemblyScript/assemblyscript) मैपिंग्स फ़ाइल इवेंट डेटा से आपके स्कीमा में परिभाषित की गई संस्थाओं में डेटा का अनुवाद करती है।

'create a subgraph' करने के लिए विस्तृत जानकारी प्राप्त करें [create a subgraph](/developing/creating-a-subgraph/)।

## सबग्राफ जीवनचक्र

यहाँ एक Subgraph के जीवनचक्र का सामान्य अवलोकन है।

![सबग्राफ जीवनचक्र](/img/subgraph-lifecycle.png)

### स्थानीय रूप से बनाएँ

महान subgraphs एक स्थानीय विकास वातावरण और यूनिट परीक्षणों के साथ शुरू होते हैं। डेवलपर्स [Graph CLI](https://github.com/graphprotocol/graph-tooling/tree/main/packages/cli), एक कमांड-लाइन इंटरफेस उपकरण का उपयोग करते हैं जो subgraphs को The Graph पर बनाने और तैनात करने के लिए है। वे [Graph TypeScript](/developing/graph-ts/) और [Matchstick](/developing/unit-testing-framework/) का उपयोग करके मजबूत subgraphs भी बना सकते हैं।

### Deploy to Subgraph Studio

एक बार परिभाषित होने के बाद, एक subgraph को [Subgraph Studio](/deploying/deploying-a-subgraph-to-studio/) में तैनात किया जा सकता है। Subgraph Studio में, आप निम्नलिखित कर सकते हैं:

- इसके स्टेजिंग वातावरण का उपयोग करें ताकि परिनियोजित Subgraph को इंडेक्स किया जा सके और इसे समीक्षा के लिए उपलब्ध कराया जा सके।
- अपने subgraph की जांच करें कि इसमें कोई indexing त्रुटियाँ नहीं हैं और यह अपेक्षानुसार काम करता है।

### नेटवर्क पर प्रकाशित करें

जब आप अपने सबग्राफ़ से संतुष्ट हों, तो आप इसे **The Graph Network** पर [प्रकाशित कर सकते हैं](/publishing/publishing-a-subgraph/)।

- यह एक ऑन-चेन क्रिया है, जो Subgraph को पंजीकृत करती है और इसे Indexer द्वारा खोजने योग्य बनाती है।
- प्रकाशित subgraphs का एक संबंधित NFT होता है, जो subgraph की स्वामित्व को परिभाषित करता है। आप NFT भेजकर [subgraph की स्वामित्व ट्रांसफर](/managing/transfer-a-subgraph/) कर सकते हैं।
- प्रकाशित सबग्राफ़ के साथ मेटाडेटा होता है, जो अन्य नेटवर्क प्रतिभागियों को उपयोगी संदर्भ और जानकारी प्रदान करता है।

### Indexing के लिए क्यूरेशन सिग्नल जोड़ें

प्रकाशित सबग्राफ़ को क्यूरेशन सिग्नल के बिना Indexer द्वारा उठाने की संभावना कम होती है। इंडेक्सिंग को प्रोत्साहित करने के लिए, आपको अपने Subgraph में सिग्नल जोड़ना चाहिए। The Graph पर सिग्नलिंग और [curating](/network/curating/) के बारे में अधिक जानें।

#### सिग्नल क्या है?

- सिग्नल वह GRT है जो एक विशेष Subgraphसे जुड़ा हुआ है और लॉक किया गया है। यह Indexer को यह संकेत देता है कि एक विशेष Subgraph को क्वेरी वॉल्यूम प्राप्त होगा और यह इसे प्रोसेस करने के लिए उपलब्ध indexing पुरस्कारों में योगदान करता है।
- यदि तीसरे पक्ष के क्यूरेटर को लगता है कि Subgraph क्वेरी वॉल्यूम उत्पन्न करने की संभावना रखता है, तो वे किसी दिए गए Subgraph पर भी सिग्नल कर सकते हैं।

### पूछताछ & एप्लीकेशन का विकास

Subgraphs on The Graph Network receive 100,000 free queries per month, after which point developers can either [pay for queries with GRT or a credit card](/billing/).

Learn more about [querying subgraphs](/querying/querying-the-graph/).

### Updating Subgraphs

To update your subgraph with bug fixes or new functionalities, initiate a transaction to point it to the new version. You can deploy new versions of your subgraphs to [Subgraph Studio](https://thegraph.com/studio/) for development and testing.

- If you selected "auto-migrate" when you applied the signal, updating the subgraph will migrate any signal to the new version and incur a migration tax.
- This signal migration should prompt Indexers to start indexing the new version of the subgraph, so it should soon become available for querying.

### Deprecating & Transferring Subgraphs

If you no longer need a published subgraph, you can [deprecate](/managing/deprecate-a-subgraph/) or [transfer](/managing/transfer-a-subgraph/) it. Deprecating a subgraph returns any signaled GRT to [Curators](/network/curating/).
