---
title: Glossar
---

- **The Graph**: Ein dezentrales Protokoll zur Indizierung und Abfrage von Daten.

- **Abfrage**: Eine Anfrage nach Daten. Im Fall von The Graph ist eine Abfrage eine Anfrage nach Daten aus einem Subgraph, die von einem Indexer beantwortet wird.

- **GraphQL**: Eine Abfragesprache für APIs und eine Laufzeitumgebung, um diese Abfragen mit Ihren vorhandenen Daten zu erfüllen. The Graph verwendet GraphQL, um Subgraphen abzufragen.

- **Endpunkt**: Eine URL, die zur Abfrage eines Subgraphen verwendet werden kann. Der Test-Endpunkt für Subgraph Studio ist `https://api.studio.thegraph.com/query/<ID>/<SUBGRAPH_NAME>/<VERSION>` und der Graph Explorer Endpunkt ist `https://gateway.thegraph.com/api/<API_KEY>/subgraphs/id/<SUBGRAPH_ID>`. Der The Graph Explorer Endpunkt wird verwendet, um Subgraphen im dezentralen Netzwerk von The Graph abzufragen.

- **Subgraph**: Eine offene API, die Daten aus einer Blockchain extrahiert, verarbeitet und so speichert, dass sie einfach über GraphQL abgefragt werden können. Entwickler können einen Subgraph erstellen, bereitstellen und auf The Graph Network veröffentlichen. Sobald der Subgraph indiziert ist, kann er von jedem abgefragt werden.

- **Indexer**: Netzwerkteilnehmer, die Indexierungsknoten betreiben, um Daten aus Blockchains zu indexieren und GraphQL-Abfragen zu bedienen.

- **Einkommensströme für Indexer**: Indexer werden in GRT mit zwei Komponenten belohnt: Rabatte auf Abfragegebühren und Rewards für die Indizierung.

  1. **Abfragegebühren-Rabatte**: Zahlungen von Subgraph-Konsumenten für die Bedienung von Anfragen im Netz.

  2. **Indizierungs-Rewards**: Die Rewards, die Indexer für die Indizierung von Subgraphen erhalten. Indexing Rewards werden durch die Neuausgabe von 3% GRT jährlich generiert.

- **Selbstbeteiligung der Indexer**: Der Betrag an GRT, den Indexer einsetzen, um am dezentralen Netzwerk teilzunehmen. Das Minimum beträgt 100.000 GRT, eine Obergrenze gibt es nicht.

- **Delegationskapazität**: Die maximale Menge an GRT, die ein Indexer von Delegatoren annehmen kann. Indexer können nur bis zum 16-fachen ihres Indexer-Eigenanteils akzeptieren, und zusätzliche Delegationen führen zu verwässerten Rewards. Ein Datenbeispiel: Wenn ein Indexer eine Selbsteinnahme von 1 Mio. GRT hat, beträgt seine Delegationskapazität 16 Mio. GRT. Indexer können jedoch ihre Delegationskapazität erhöhen, indem sie ihre Selbstbeteiligung erhöhen.

- **Upgrade Indexer**: Ein Indexer, der als Fallback für Subgraph-Abfragen dient, die von anderen Indexern im Netzwerk nicht bedient werden. Der Upgrade Indexer ist nicht konkurrenzfähig zu anderen Indexern.

- **Delegator**: Netzwerkteilnehmer, die GRT besitzen und ihre GRT an Indexer delegieren. Dadurch können Indexer ihren Anteil an Subgraphen im Netzwerk erhöhen. Im Gegenzug erhalten Delegatoren einen Teil der Indexing Rewards, die Indexer für die Bearbeitung von Subgraphen erhalten.

- **Delegationssteuer**: Eine 0,5%ige Gebühr, die von Delegatoren gezahlt wird, wenn sie GRT an Indexer delegieren. Die GRT, die zur Zahlung der Gebühr verwendet werden, werden verbrannt.

- **Kurator**: Netzwerkteilnehmer, die qualitativ hochwertige Subgraphen identifizieren und GRT für sie im Austausch für Kurationsanteile signalisieren. Wenn Indexer Abfragegebühren für einen Subgraph einfordern, werden 10% an die Kuratoren dieses Subgraphen verteilt. Es besteht eine positive Korrelation zwischen der Höhe der signalisierten GRT und der Anzahl der Indexer, die einen Subgraph indexieren.

- **Kurationssteuer**: Eine Gebühr von 1 %, die von Kuratoren gezahlt wird, wenn sie GRT auf Subgraphen melden. Die GRT, die zur Zahlung der Gebühr verwendet wird, wird verbrannt.

- **Datenverbraucher**: Jede Anwendung oder jeder Benutzer, der einen Subgraph abfragt.

- **Subgraphen-Entwickler**: Ein Entwickler, der einen Subgraph für das dezentrale Netzwerk von The Graph erstellt und bereitstellt.

- **Subgraph Manifest**: Eine YAML-Datei, die das GraphQL-Schema, die Datenquellen und andere Metadaten des Subgraphen beschreibt. Ein Beispiel ist [hier](https://github.com/graphprotocol/example-subgraph/blob/master/subgraph.yaml).

- **Epoche**: Eine Zeiteinheit innerhalb des Netzes. Derzeit entspricht eine Epoche 6.646 Blöcken oder etwa 1 Tag.

- **Zuteilung**: Ein Indexer kann seinen gesamten GRT-Einsatz (einschließlich des Einsatzes der Delegatoren) Subgraphen zuweisen, die im dezentralen Netzwerk von The Graph veröffentlicht wurden. Zuteilungen können verschiedene Status haben:

  1. **Aktiv**: Eine Zuordnung gilt als aktiv, wenn sie onchain erstellt wird. Dies wird als Öffnen einer Zuordnung bezeichnet und zeigt dem Netzwerk an, dass der Indexer aktiv indiziert und Abfragen für einen bestimmten Subgraph bedient. Aktive Zuweisungen sammeln Rewards für die Indizierung, die proportional zum Signal auf dem Subgraph und der Menge des zugewiesenen GRT sind.

  2. **Geschlossen**: Ein Indexer kann die aufgelaufenen Rewards für einen bestimmten Subgraph beanspruchen, indem er einen aktuellen und gültigen Proof of Indexing (POI) einreicht. Dies wird als Schließen einer Zuordnung bezeichnet. Eine Zuordnung muss mindestens eine Epoche lang offen gewesen sein, bevor sie geschlossen werden kann. Die maximale Zuordnungsdauer beträgt 28 Epochen. Lässt ein Indexer eine Zuordnung länger als 28 Epochen offen, wird sie als veraltete Zuordnung bezeichnet. Wenn sich eine Zuordnung im Zustand **Geschlossen** befindet, kann ein Fisherman immer noch einen Disput eröffnen, um einen Indexer wegen der Bereitstellung falscher Daten anzufechten.

- **Subgraph Studio**: Eine leistungsstarke App zum Erstellen, Bereitstellen und Veröffentlichen von Subgraphen.

- **Fischer**: Eine Rolle innerhalb des The Graph Network, die von Teilnehmern eingenommen wird, die die Genauigkeit und Integrität der von Indexern gelieferten Daten überwachen. Wenn ein Fisherman eine Abfrage-Antwort oder einen POI identifiziert, den er für falsch hält, kann er einen Disput gegen den Indexer einleiten. Wenn der Streitfall zu Gunsten des Fischers entschieden wird, verliert der Indexer 2,5 % seines Eigenanteils. Von diesem Betrag erhält der Fischer 50 % als Belohnung für seine Wachsamkeit, und die restlichen 50 % werden aus dem Verkehr gezogen (verbrannt). Dieser Mechanismus soll die Fischer dazu ermutigen, die Zuverlässigkeit des Netzwerks aufrechtzuerhalten, indem sichergestellt wird, dass die Indexer für die von ihnen gelieferten Daten verantwortlich gemacht werden.

- **Schlichter**: Schlichter sind Netzwerkteilnehmer, die im Rahmen eines Governance-Prozesses ernannt werden. Die Rolle des Schiedsrichters besteht darin, über den Ausgang von Streitigkeiten bei Indizierungen und Abfragen zu entscheiden. Ihr Ziel ist es, den Nutzen und die Zuverlässigkeit von The Graph Network zu maximieren.

- **Kürzung**: Indexer können für die Bereitstellung eines falschen POI oder für die Bereitstellung ungenauer Daten um ihre selbst gesetzten GRT gekürzt werden. Der Prozentsatz des Slashings ist ein Protokollparameter, der derzeit auf 2,5% des Eigenanteils eines Indexers festgelegt ist. 50 % der gekürzten GRT gehen an den Fischer, der die ungenauen Daten oder den falschen POI bestritten hat. Die anderen 50% werden verbrannt.

- **Indizierungs-Rewards**: Die Rewards, die Indexer für die Indizierung von Subgraphen erhalten. Indexing Rewards werden in GRT verteilt.

- **Delegation Rewards**: Die Rewards, die Delegatoren für die Delegierung von GRT an Indexer erhalten. Delegations-Rewards werden in GRT verteilt.

- **GRT**: The Graph's Work Utility Token. GRT bietet den Netzwerkteilnehmern wirtschaftliche Anreize für ihren Beitrag zum Netzwerk.

- **Nachweis der Indizierung (POI)**: Wenn ein Indexer seine Zuteilung abschließt und seine angesammelten Rewards für einen bestimmten Subgraph beanspruchen möchte, muss er einen gültigen und aktuellen POI vorlegen. Fischer können den von einem Indexer angegebenen POI anfechten. Ein Streit, der zugunsten des Fischers entschieden wird, führt zur Streichung des Indexers.

- **Graph Node**: Graph Node ist die Komponente, die Subgraphen indiziert und die resultierenden Daten für Abfragen über eine GraphQL-API zur Verfügung stellt. Als solche ist sie von zentraler Bedeutung für den Indexer-Stack, und der korrekte Betrieb von Graph Node ist entscheidend für den erfolgreichen Betrieb eines Indexers.

- **Indexer-Agent**: Der Indexer-Agent ist Teil des Indexer-Stacks. Er erleichtert die Interaktionen des Indexers in der Kette, einschließlich der Registrierung im Netzwerk, der Verwaltung von Subgraph-Bereitstellungen an seine(n) Graph Node(s) und der Verwaltung von Zuweisungen.

- **The Graph Client**: Eine Bibliothek für den Aufbau von GraphQL-basierten Dapps auf dezentralisierte Weise.

- **Graph Explorer**: Eine App, mit der Netzwerkteilnehmer Subgraphen erkunden und mit dem Protokoll interagieren können.

- **Graph CLI**: Ein Befehlszeilenschnittstellen-Tool zum Erstellen und Bereitstellen von The Graph.

- **Abkühlphase**: Die Zeit, die verbleibt, bis ein Indexer, der seine Delegationsparameter geändert hat, dies wieder tun kann.

- **L2 Transfer Tools**: Smart Contracts und UI, die es Netzwerkteilnehmern ermöglichen, netzwerkbezogene Vermögenswerte vom Ethereum Mainnet auf Arbitrum One zu übertragen. Netzwerkteilnehmer können delegiertes GRT, Subgraphen, Kurationsanteile und den Eigenanteil des Indexers übertragen.

- **Aktualisierung eines Subgraphen**: Der Prozess der Freigabe einer neuen Version eines Subgraphen mit Aktualisierungen des Manifests, des Schemas oder der Mappings des Subgraphen.

- **Migrieren**: Der Prozess, bei dem Kurationsanteile von einer alten Version eines Subgraphs auf eine neue Version eines Subgraphs übertragen werden (z. B. wenn v0.0.1 auf v0.0.2 aktualisiert wird).
