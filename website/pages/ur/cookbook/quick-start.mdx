---
title: فورا شروع کریں
---

یہ گائیڈ آپ کو سب گراف اسٹوڈیو یا [ہوسٹڈ سروس](#hosted-service) میں اپنے سب گراف کو شروع کرنے، تخلیق کرنے اور تعینات کرنے کے طریقہ کے بارے میں فوری طور پر لے جائے گا.

اس بات کو یقینی بنائیں کہ آپ کا سب گراف کسی [تعاون یافتہ نیٹ ورک](/developing/supported-networks) سے ڈیٹا کو ترتیب دے رہا ہے.

## سب گراف اسٹوڈیو

### ۱. گراف CLI انسٹال کریں

گراف CLI جاوا اسکریپٹ میں لکہی گئ ہے اور اسے استعمال کرنے کے لیے آپ کو یا تو `npm` یا ` Yarn` انسٹال کرنے کی ضرورت ہوگی.

```sh
# NPM
$ npm install -g @graphprotocol/graph-cli

# Yarn
$ yarn global add @graphprotocol/graph-cli
```

### ۲. اپنا سب گراف شروع کریں

- اپنے سب گراف کو موجودہ کنٹریکٹ سے شروع کریں.

```sh
graph init --studio <SUBGRAPH_SLUG>
```

- آپ کا سب گراف سلگ آپ کے سب گراف کا شناخت کنندہ ہے۔ CLI ٹول آپ کو سب گراف بنانے کے مراحل سے گزرے گا، جیسا کہ کنٹریکٹ ایڈریس، نیٹ ورک، وغیرہ جیسا کہ آپ نیچے اسکرین شاٹ میں دیکھ سکتے ہیں.

![سب گراف کمانڈ](/img/Subgraph-Slug.png)

> نوٹ: وقت بچانے کے لیے اوپر والی کمانڈ میں `--index-events` شامل کرنے پر غور کریں۔ یہ سب گراف کو اسکیما میں موجود اداروں کے ساتھ بوٹسٹریپ کرتا ہے اور ہر خارج ہونے والے ایونٹ کے لیے سادہ میپنگ کرتا ہے.

### ۳. اپنا سب گراف لکھیں

پچھلی کمانڈز ایک سکیفولڈ سب گراف بناتی ہیں جسے آپ اپنے سب گراف کی تعمیر کے لیے نقطہ آغاز کے طور پر استعمال کر سکتے ہیں۔ سب گراف میں تبدیلی کرتے وقت، آپ بنیادی طور پر تین فائلوں کے ساتھ کام کریں گے:

- مینی فیسٹ (subgraph.yaml) - مینی فیسٹ اس بات کی وضاحت کرتا ہے کہ آپ کے سب گراف کس ڈیٹا سورسز کو انڈیکس کریں گے.
- سکیما (schema.graphql) - گراف کیو ایل سکیما اس بات کی وضاحت کرتا ہے کہ آپ سب گراف سے کون سا ڈیٹا حاصل کرنا چاہتے ہیں.
- اسمبلی اسکرپٹ میپنگ (mapping.ts) - یہ وہ کوڈ ہے جو آپ کے ڈیٹا سورس سے ڈیٹا کو اسکیما میں بیان کردہ اداروں میں ترجمہ کرتا ہے.

مزید معلومات کے کیسے سب گراف لکہا جاۓ، [سب گراف بنانا](/developing/creating-a-subgraph) دیکہیں.

### ۴. سب گراف سٹوڈیو پر تعینات کریں

- سب گراف سٹوڈیو پر جائیں [https://thegraph.com/studio/](https://thegraph.com/studio/) اور اپنا والیٹ کونیکٹ کریں.
- "create" کو دباییں اور وہ سب گراف سلگ درج کریں جو آپ نے مرحلہ 2 میں استعمال کیا ہے.
- یہ کمانڈس سب گراف فولڈر میں چلائیں

```sh
$ graph codegen
$ graph build
```

- اپنے سب گراف کی تصدیق اور اسے تعینات کریں. تعیناتی کی کلید آپ کو سب گراف پیج پر ملے گی جو سب گراف سٹوڈیو میں موجود ہے.

```sh
$ graph auth --studio <DEPLOY_KEY>
$ graph deploy --studio <SUBGRAPH_SLUG>
```

- آپ سے ورزن لیبل پوچھا جاۓ گا. اپنے ورژن کے نام رکھنے کے لیے درج ذیل کنونشنز کو استعمال کرنے کی سختی سے سفارش کی جاتی ہے. مثال: `0.0.1`, `v1`, `version1`

### ۵. اپنے لوگس چیک کریں

لوگس کو آپ کو بتانا چاہئے اگر کہیں پرغلطیاں ہیں. اگر آپ کا سب گراف ناکام ہو رہا ہے، تو آپ [GraphiQL پلے گراؤنڈ](https://graphiql-online.com/) کا استعمال کر کے سب گراف کی صحت کے بارے میں کیوری کر سکتے ہیں. [یہ اختتامی نقطہ](https://api.thegraph.com/index-node/graphql) استعمال کریں. نوٹ کریں کہ آپ نیچے دیے گئے کیوری سے فائدہ اٹھا سکتے ہیں اور اپنے سب گراف کے لیے اپنی تعیناتی ID درج کر سکتے ہیں۔ اس صورت میں، `Qm...` تعیناتی ID ہے (جو **تفصیلات** کے تحت سب گراف کے صفحہ پر واقع ہوسکتی ہے)۔ ذیل کا کیوری آپ کو بتائے گا کہ سب گراف کب ناکام ہوجاتا ہے تاکہ آپ اس کے مطابق ڈیبگ کرسکیں:

```sh
{
  indexingStatuses(subgraphs: ["Qm..."]) {
    node
    synced
    health
    fatalError {
      message
      block {
        number
        hash
      }
      handler
    }
    nonFatalErrors {
      message
      block {
        number
        hash
      }
      handler
    }
    chains {
      network
      chainHeadBlock {
        number
      }
      earliestBlock {
        number
      }
      latestBlock {
        number
      }
      lastHealthyBlock {
        number
      }
    }
    entityCount
  }
}
```

### ۶. اپنے سب گراف کو کیوری کریں

آپ اپنے سب گراف کا کیوری [ان ہدایات](/querying/querying-the-graph) پر عمل کر کے کر سکتے ہیں. آپ اپنے ڈیپ سے کیوری کر سکتے ہیں اگر آپ کے پاس اپنی API کلید مفت، ریٹ محدود عارضی کیوری URL کے ذریعے نہیں ہے جسے ترقی اور سٹیجنگ کے لیے استعمال کیا جا سکتا ہے۔ آپ فرنٹ اینڈ ایپلیکیشن سے سب گراف سے کیوری کرنے کے بارے میں اضافی ہدایات [یہاں](/querying/querying-from-an-application) پڑھ سکتے ہیں.

## ھوسٹڈ سروس

### ۱. گراف CLI انسٹال کریں

گراف CLI ایک npm پیکج ہے اور آپ کو `npm` یا `yarn` کو انسٹال کرنا پڑے گا اس کو استعمال کرنے کے لۓ.

```sh
# NPM
$ npm install -g @graphprotocol/graph-cli

# Yarn
$ yarn global add @graphprotocol/graph-cli
```

### ۲. اپنا سب گراف شروع کریں

- اپنے سب گراف کو موجودہ کنٹریکٹ سے شروع کریں.

```sh
$ graph init --product hosted-service --from-contract <Address>
```

- آپ سے سب گراف کا نام پوچہا جاۓ گا. `<Github>/<Subgraph Name>` فارمیٹ ہے. مثال: `graphprotocol/examplesubgraph`

- اگر آپ ایک مثال سے شروع کرنا چاہتے ہیں, نیچے دی گئ کمانڈ کو چلائیں:

```sh
$ graph init --product hosted-service --from-example
```

- مثال کی صورت میں، سب گراف Dani گرانٹ کے گریوٹی کنٹریکٹ پر مبنی ہے جو صارف کے اوتاروں کا انتظام کرتا ہے اور `NewGravatar` یا `UpdateGravatar` ایونٹس کو خارج کرتا ہے جب بھی اوتار بنائے یا اپ ڈیٹ ہوتے ہیں.

### ۳. اپنا سب گراف لکھیں

پچھلی کمانڈ نے ایک سہارہ بنایا ہوگا جہاں سے آپ اپنا سب گراف بنا سکتے ہیں۔ سب گراف میں تبدیلی کرتے وقت، آپ بنیادی طور پر تین فائلوں کے ساتھ کام کریں گے:

- مینی فیسٹ (subgraph.yaml) - مینی فیسٹ یہ بیان کرتا ہے کے کونسے دیٹا سورسز آپ کا سب گراف انڈیکس کرے گا
- سکیما (schema.graphql) - گراف کیو ایل سکیما بیان کرتا ہے کونسا ڈیٹا آپ سب گراف سے نکالنا چاہتے ہیں
- اسمبلی اسکریپٹ میپنگ (mapping.ts) - یہ وہ کوڈ ہے جو آپ کے ڈیٹا سورسز سے ڈیٹا کو اسکیما میں بیان کردہ اداروں میں ترجمہ کرتا ہے

مزید معلومات کے کیسے سب گراف لکہا جاۓ، [سب گراف بنانا](/developing/creating-a-subgraph) دیکہیں.

### ۴. اپنے سب گراف کو تعینات کریں

- اپنا گٹ ہب اکاؤنٹ استعمال کرتے ہوۓ [ہوسٹڈ سروس](https://thegraph.com/hosted-service/) میں سائن ان کریں
- "سب گراف شامل کریں" کو دبائیں اور ضروری معلومات کو پر کریں. اسی سب گراف کا نام استعمال کریں جیسا کہ مرحلہ 2 میں ہے.
- `codegen` کو چلائیں جو سب گراف فولڈر میں ہے

```sh
 # NPM
$ npm run codegen

# Yarn
$ yarn codegen
```

- اپنے رسائ کے ٹوکن کو شامل کریں اور اپنے سب گراف کو تعینات کر دیں. یہ رسائ کا ٹوکن ہوسٹڈ سروس میں آپ کے ڈیش بورڈ پر ملے گا.

```sh
$ graph auth --product hosted-service <ACCESS_TOKEN>
$ graph deploy --product hosted-service <GITHUB_USER>/<SUBGRAPH NAME>
```

### ۵. اپنے لوگس چیک کریں

### ۶. اپنے سب گراف کو کیوری کریں

ہوسٹڈ سروس پر اپنے سب گراف کو کیوری کرنے کے لۓ [ان ہدایات](/querying/querying-the-hosted-service) پر عمل کریں.
