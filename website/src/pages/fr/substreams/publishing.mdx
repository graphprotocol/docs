Apprenez à publier un package Substreams sur le [Registre Substreams] (https://substreams.dev).

## Aperçu

### Qu'est-ce qu'un package ?

Un package Substreams est un fichier binaire précompilé qui définit les données spécifiques que vous souhaitez extraire de la blockchain, similaire au fichier `mapping.ts` dans les Subgraphs traditionnels.

## Publier un package

### Prérequis

- La CLI de Substreams doit être installé.
- Vous devez avoir un package Substreams (`.spkg`) que vous voulez publier.

### Étape 1 : Exécuter la commande `substreams publish`

1. Dans un terminal de ligne de commande, lancez `substreams publish <Votre-PACKAGE>.spkg`.

2. Si vous n'avez pas de jeu de jetons sur votre ordinateur, naviguez vers `https://substreams.dev/me`.

![obtenir un jeton](/img/1_get-token.png)

### Étape 2 : Obtenir un jeton dans le registre de Substreams

1. Dans le registre Substreams, connectez-vous avec votre compte GitHub.

2. Créez un nouveau jeton et copiez-le dans un endroit sûr.

![nouveau jeton](/img/2_new_token.png)

### Étape 3 : Authentification dans l'interface de gestion de Substreams

1. De retour dans le CLI de Substreams, collez le jeton généré précédemment.

![collez le jeton](/img/3_paste_token.png)

2. Enfin, confirmez que vous souhaitez publier le package.

![confirmer](/img/4_confirm.png)

C'est Terminé ! Vous avez réussi à publier un package dans le registre de Substreams. Vous avez publié avec succès un package dans le registre Substreams.

![succès](/img/5_success.png)
