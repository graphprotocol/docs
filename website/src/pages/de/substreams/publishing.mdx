Erfahren Sie, wie Sie ein Substreams-Paket in der [Substreams Registry] (https://substreams.dev) veröffentlichen.

## Überblick

### Was ist ein Paket?

Ein Substreams-Paket ist eine vorkompilierte Binärdatei, die die spezifischen Daten definiert, die Sie aus der Blockchain extrahieren möchten, ähnlich wie die Datei `mapping.ts` in traditionellen Subgraphen.

## Veröffentlichung eines Pakets

### Voraussetzungen

- Sie müssen die Substreams CLI installiert haben.
- Sie müssen ein Substreams-Paket (`.spkg`) haben, das Sie veröffentlichen wollen.

### Schritt 1: Führen Sie den Befehl `substreams publish` aus

1. Führen Sie in einem Befehlszeilen-Terminal die Datei `substreams publish <YOUR-PACKAGE>.spkg` aus.

2. Wenn Sie keinen Token auf Ihrem Computer haben, navigieren Sie zu `https://substreams.dev/me`.

![get token](/img/1_get-token.png)

### Schritt 2: Erhalten Sie ein Token in der Substrats-Registrierung

1. Melden Sie sich in der Substreams Registry mit Ihrem GitHub-Konto an.

2. Erstellen Sie einen neuen Token und kopieren Sie ihn an einen sicheren Ort.

![neues Token](/img/2_new_token.png)

### Schritt 3: Authentifizierung in der Substreams-CLI

1. Zurück in der Substreams-CLI fügen Sie das zuvor generierte Token ein.

![Token einfügen](/img/3_paste_token.png)

2. Bestätigen Sie abschließend, dass Sie das Paket veröffentlichen möchten.

![bestätigen](/img/4_confirm.png)

Das war's! Sie haben erfolgreich ein Paket in der Substreams-Registrierung veröffentlicht.

![Erfolg](/img/5_success.png)
