こちらのページでは、Graph Assistantを利用する開発者から寄せられる、よくある質問をまとめました。

1. Graph Assistant とは何ですか？

Graph Assistant は、AI を活用したチャットインターフェイスで、平易な英語でブロックチェーンデータを問い合わせることができます。内部的には、ユーザーの質問を Subgraph や Token API に対する GraphQL 呼び出しに変換し、コーディングを一切することなく結果を返します。

2. エージェント アプリケーションとは何ですか？

エージェント アプリケーション は、AI の推論機能と外部データソースや API を組み合わせて、ユーザーに代わって自律的な行動をとるアプリケーションです。この場合、Graph Assistant は、ユーザーの自然言語のプロンプトを解釈し、GraphQL クエリを構築・実行し、結果を整形する「エージェント」として機能するため、ユーザー自身がクエリを作成・管理する必要がありません。

3. Graph Assistant はどのネットワークとデータに対応していますか？

Graph Assistant は、複数のチェーンにわたる Subgraph にアクセスできます。Subgraph が The Graph Network に公開され、インデックスが作成されていれば、Assistant は自動的にクエリをルーティングできます。また、Assistant は Ethereum Mainnet、BSC、Unichain、Arbitrum-One、Optimism、Polygon の Token API にもアクセスできます。

4. 利用制限や割り当てはありますか？はい、1時間あたり10件の質問という制限があります。

5. Assistant が質問を解釈できない場合はどうなりますか？

Assistant がユーザーの意図を正確に解析できなかった場合（例えば、トークン名が曖昧、文脈が不足、または非対応のリクエストの場合など）、明確化を求めるプロンプトが表示されます。例えば、「XYZ は Ethereum と Arbitrum のどちらを指しますか？」や、「このクエリの日付範囲を指定してください」といった質問が表示される場合があります。解析エラーが続く場合は、正しいトークンシンボルを使用しているか、ネットワークや時間枠を明確に指定しているかを確認してください。
